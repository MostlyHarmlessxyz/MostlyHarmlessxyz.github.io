```
你现在运行在一台 Windows 主机上。请作为“Linux 安装迁移助手”，为当前这台机器自动完成硬件环境采集、Linux 发行版可行性调研、安装方案设计、文件下载整理和安装 Guide 文档编写。

我的目标是：

1. 优先尝试安装 Arch Linux。
2. 使用 U 盘上的 Ventoy 启动安装镜像。
3. 当前机器大致配置可能包括：

   * CPU：Intel Core i5-13400
   * 显卡：Intel Arc A750
   * 无线网卡：Intel AX211
   * 主板：精粤 ITX 主板
4. 如果 Arch Linux 不适合作为首选，请给出更优替代方案，例如 Fedora Workstation、EndeavourOS、Ubuntu 24.04 LTS 等，并说明理由。
5. 请重点关注 Intel Arc A750、Intel AX211、精粤 ITX 主板、Resizable BAR、UEFI、Secure Boot、Ventoy 兼容性、Wi-Fi 驱动、Mesa/Vulkan、linux-firmware、安装后图形桌面可用性。

请严格按照下面流程执行。

一、创建工作目录

在当前用户的桌面上创建目录：

Linux_Install_Research

目录结构如下：

Linux_Install_Research/
├── hardware_report/
├── downloads/
│   ├── iso/
│   ├── ventoy/
│   ├── drivers_firmware/
│   └── tools/
├── docs/
├── scripts/
└── logs/

所有采集结果、下载文件、网页资料摘要、最终文档都放到这个目录下。

二、采集当前 Windows 硬件与系统信息

请使用 PowerShell 命令自动采集硬件信息，并保存为 txt/json/csv 文件。

至少采集：

1. Windows 版本、系统架构、启动模式。
2. CPU 型号、核心线程、虚拟化支持。
3. 主板厂商、型号、BIOS 版本、BIOS 日期。
4. 显卡型号，包括核显和独显。
5. 网卡型号，包括有线网卡和无线网卡。
6. 声卡型号。
7. 磁盘信息，包括每块磁盘的型号、容量、分区表类型 GPT/MBR、当前分区布局。
8. 内存容量、频率、插槽信息。
9. 是否为 UEFI 启动。
10. Secure Boot 状态。
11. TPM 状态。
12. BitLocker 状态。
13. 当前磁盘剩余空间。
14. 当前是否开启 Hyper-V、WSL、虚拟化平台等。
15. PCI 设备列表，尤其关注 Intel Arc A750、Intel AX211、有线网卡芯片。

可使用但不限于以下命令：

* systeminfo
* msinfo32 /report
* Get-ComputerInfo
* Get-CimInstance Win32_Processor
* Get-CimInstance Win32_BaseBoard
* Get-CimInstance Win32_BIOS
* Get-CimInstance Win32_VideoController
* Get-CimInstance Win32_NetworkAdapter
* Get-CimInstance Win32_DiskDrive
* Get-CimInstance Win32_PhysicalMemory
* Get-Disk
* Get-Partition
* Confirm-SecureBootUEFI
* Get-BitLockerVolume
* bcdedit
* pnputil /enum-devices
* wmic path win32_videocontroller get name,pnpdeviceid
* wmic path win32_networkadapter get name,pnpdeviceid

请把执行过的所有命令保存到：

scripts/collect_hardware.ps1

并把命令输出保存到：

hardware_report/

三、自动分析硬件兼容性

请根据采集结果，自动判断这台机器安装 Linux 的风险点。

重点分析：

1. Intel Core i5-13400：

   * Linux 内核支持情况。
   * 大小核调度是否需要新内核。
   * 核显 UHD 730 是否可用。

2. Intel Arc A750：

   * 需要的最低 Linux kernel 版本。
   * 需要的 Mesa 版本。
   * 是否需要 linux-firmware。
   * Vulkan/DXVK/VAAPI/视频硬解支持情况。
   * 是否需要开启 Resizable BAR。
   * Arch Linux、Fedora、Ubuntu 24.04 对 Arc A750 的支持差异。
   * 是否建议使用 X11 或 Wayland。
   * 安装后如何验证：

     * lspci -k
     * glxinfo -B
     * vulkaninfo
     * vainfo
     * dmesg | grep -iE "i915|xe|guc|huc|drm"

3. Intel AX211：

   * Linux 是否支持。
   * iwlwifi 驱动支持情况。
   * 需要的 linux-firmware。
   * 是否可能因为固件缺失导致安装阶段没有 Wi-Fi。
   * 是否建议准备手机 USB 网络共享或有线网兜底。

4. 精粤 ITX 主板：

   * BIOS 兼容性风险。
   * 是否需要更新 BIOS。
   * BIOS 里需要检查的选项：

     * UEFI Boot
     * CSM Disabled
     * Secure Boot
     * Above 4G Decoding
     * Resizable BAR
     * SATA Mode AHCI
     * XMP
     * ASPM
   * 便宜主板可能存在的 Linux 风险：

     * 睡眠唤醒异常
     * 网卡驱动问题
     * 声卡问题
     * BIOS 选项不完整
     * 风扇曲线/温控异常

5. Ventoy：

   * Arch Linux ISO 是否适合直接用 Ventoy 启动。
   * 是否存在 Secure Boot 兼容问题。
   * Ventoy 分区格式建议。
   * 是否建议关闭 Secure Boot。
   * 是否需要准备一个传统 Rufus/dd 写入的备用 U 盘方案。
   * 如果 Ventoy 启动 Arch ISO 失败，给出替代方案。

五、选择发行版并给出判断

请根据硬件和目标，给出以下发行版的对比：

1. Arch Linux
2. EndeavourOS
3. Fedora Workstation
4. Ubuntu 24.04 LTS
5. openSUSE Tumbleweed，如果有必要也可以加入

比较维度：

* 对 Intel Arc A750 的支持
* 对 Intel AX211 的支持
* 内核和 Mesa 新旧
* 安装难度
* 滚动更新风险
* 游戏/Proton/DXVK 支持
* 开发环境支持
* 系统稳定性

如果 Arch Linux 风险过高，请明确建议我改用 Fedora Workstation 或 EndeavourOS，并解释原因。

六、下载并整理所需文件

请自动下载或整理下载链接。优先下载到：

downloads/

需要准备：

1. Arch Linux 最新 ISO。
2. 如果建议 Fedora，也下载 Fedora Workstation 最新 ISO。
3. 如果建议 EndeavourOS，也下载 EndeavourOS 最新 ISO。
4. Ventoy 最新 Windows 版本。
5. Rufus 最新便携版，作为 Ventoy 失败时的备用工具。
6. Intel Wi-Fi AX211 相关 linux-firmware 信息。如果 Linux 固件无法单独以 Windows 方式准备，请在 Guide 中说明安装时通过 linux-firmware 软件包解决。
7. Intel Arc A750 相关说明文档或链接。
8. Windows 下备份网卡驱动的方法说明。
9. 可选：7-Zip、Hash 校验工具。

下载要求：

1. 只从官方网站、GitHub Release 官方仓库或可信镜像站下载。
2. 对 ISO 文件尽可能下载 SHA256 校验文件或签名文件。
3. 生成一个 downloads/README.md，记录每个文件：

   * 文件名
   * 下载来源
   * 版本
   * SHA256
   * 用途
   * 是否必须
4. 如果无法直接下载，请保存官方下载页面链接，不要使用不明第三方站点。
5. 下载后计算本地 SHA256，并保存到：
   downloads/SHA256SUMS.txt
```
